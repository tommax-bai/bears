## Why

当前仓库缺少一个适合手机访问的留言板方案，也没有对应的内容审核后台。现在先定义一套移动优先、低维护成本、可控风控的产品规格，可以让后续实现直接围绕清晰的交互与治理边界展开。

## What Changes

- 新增一个移动优先的公开留言板，支持匿名或轻量昵称留言、列表浏览、发布时间展示与基础反垃圾校验。
- 新增留言发布状态流转，覆盖待审核、已发布、已拒绝、已屏蔽等后台可管理状态。
- 新增一个适合手机与桌面混合使用的管理后台，支持审核、搜索、筛选、批量处理和简单统计。
- 为留言板定义内容安全与体验约束，包括字数限制、敏感词拦截、频率限制、软删除与审计记录。

## Capabilities

### New Capabilities
- `mobile-message-board`: 面向手机访问者的留言浏览与发布体验，包括输入校验、提交反馈和移动端列表呈现。
- `message-board-admin`: 面向运营人员的留言审核后台，包括状态管理、筛选检索、批量审核和操作留痕。

### Modified Capabilities

None.

## Impact

- 影响前端信息架构，需要同时覆盖移动端留言板与后台管理界面。
- 影响后端 API 设计，需要新增留言提交、查询、审核与风控接口。
- 影响数据模型，需要定义留言内容、展示状态、审核记录与限流字段。
- 影响运营流程，需要明确审核规则、屏蔽策略与异常内容处理方式。
